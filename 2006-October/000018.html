Hi,

I just committed a functional version of GSR into subversion. 

GSR, the acronym means "Gtk Swaml Reader", is a prototype in what I'm
working these days to test some conclusions around SWAML.

It reads a mail list, given an URI, published by SWAML and it loads into
a graph. Then user can read the messages ordered by thread, besides to
make searches by terms and dates.

I made some tests and it works enough well. For example, it is able to
load a mailing list with 9.245 messages loading 122.554 triples in
memory. But I would like that you gave me your opinions about the
prototype.

You can obtain it from subversion, just checkout it:

	http://svn.berlios.de/svnroot/repos/swaml/trunk/prototypes/gsr

On README file you'll find the dependencies list (basicly PyGTK); all
works in GNU/Linux and Windows at least.

And it's very easy to use:

	python2.4 gsr.py [uri]

For example using the demo available at Berlios (be pacient with network
speed):

	python2.4 gsr.py http://swaml.berlios.de/demo/index.rdf

If you found any bug, please tell me about it.
