We are now defining a methodology to analyze each information source we
have detected in FLOSS projects. The actual draft for it is:

1. Create a card for the information source from D4.1.3, D4.2.1 and
D4.2.2 including: 
              * D4.1.3: Description, Format and licenses
              * D4.2.1: Ontologies that model the source
              * D4.2.2: Tools to manage the source
     2. Describe the general architecture with the information flow in
        the source.
     3. Research the tools for gathering the information from the
        information source (maybe some from D4.2.2).
     4. Describe the information source selected format and the mapping
        to the ontology concepts.
     5. Automatic semantic annotation for the source.
     6. Describe tools modification for adding semantic information.
     7. Describe the final semantic information model that will be
        stored in the semantic store and the relations with ontologies.
     8. Propose an implementation plan to develop and integrate the
        tools in QualiPSo A4 Semantic Framework.
     9. Execute the plan in an iterative way.


In this methodology SWAML could be used for mailing lists in 3, 4, 5 and
7. One of our main goals is at the end to have a semantic search
interface. For that we are researching to have SPARQL endpoints for all
the information sources contents. Also, we are researching reasoning
engines for more advanced information finding. Right now I am not sure
how far SWAML is in this areas.
