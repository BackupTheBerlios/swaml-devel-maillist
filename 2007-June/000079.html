> right? Do you have in mind the technologies you are going to use to
> implement it? I suppose you just need to receive the web service query
> and use RDFLib SPARQL implementation to do the query. Then you need to
> return the results from RDFLib SPARQL in the format the web service will

Unfortunately it seems that there is no implementation of the SPARQL
endpoint machinery in Python at the moment, neither the client nor the
server. As you said, an SPARQL endpoint is just a web service (either
SOAP-based or HTTP-based). I don't think we can afford building a good
Python implementation of SPARQL endpoints, although I admit it would be
an interesting project by itself! Therefore, we are studying other
options, maybe interfacing with other languages like Java.
