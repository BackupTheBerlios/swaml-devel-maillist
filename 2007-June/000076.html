Hi, Alvaro, all,

Yes, we are aware that there is a problem with memory comsumption. In
fact, SWAML reads the whole mbox file into memory, which is clearly
unacceptable for large mailing lists. We will explore some paths to
avoid this. Probably it shouldn't be difficult to process the mbox file
as a stream, and retaining in memory just the information required to
wire the links between messages.

> I have seen also that you have added "apps/prototypes/sparql-endpoint".
> Do you plan to start working in the sparql-endpoint then? Do you have a
> roadmap for it?

Actually, we are working on this now. Our idea is to add to Buxon the
feature of being able to query remote SPARQL endpoints (instead of
fetching the RDF files). At the same time, we are trying to add an
endpoint server to SWAML.
