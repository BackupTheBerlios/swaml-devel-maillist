Hi everybody,

I know that we don't use too much this list these months, but it will be
interesting your feedback for this question.

SIOC [1], an acronym that means «Semantically Interlinked Online
Communities», is a ontology that provides main concepts and properties
to describe the information from online community sites. And a mailing
list agrees with a forum definition. Then, SIOC has some relationship
with SWAML.

These days they are preparing a W3C member submission of the SIOC
ontology and specification [2]. Luis Polo has sent some interesting
notes [3] about it.

SWAML ontology [4] are completely compatible with SIOC [5]. Then I think
we have three ways:

	* Ignore SIOC, and continue using only FOAF and DC
	* Try to rewrite SWAML ontology to adapt it to SIOC
	* Devel a module to serialize SIOC using the actual core

I think that the two last ones are the most interesting for the project,
but I would appreciate so much your feedback.
