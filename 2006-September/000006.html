That's fine. Remember you are not alone ;-)

These are some suggestions:

* "suscriptors" is not an English word. Replace it by "subscribers" or
"members".

* I'm not sure if you are properly using "dc:publisher" (="SWAML").
SWAML is, in fact, the software tool used by the publisher, but not the
publisher itself.

* What is the precise meaning of "dc:date" property in sioc:Forum
instances? Maybe it is the date of the last update?

* As a good consequence of using SIOC, the distinction between sioc:User
and foaf:Person is clearer now than before. In this context, it is clear
that SWAML does not output FOAF data, and it is not until a later phase
that SIOC data is augmented with FOAF data using an external tool (idea:
can this external tool be used over SIOC data from other sources --not
SWAML--, for example, blog or forums data?)

* As you probably have noticed, swaml:Message and its swaml:* properties
are still used. I suppose you are working on this area, aren't you?

* I think you should write a document about the relations between your
ontology (swaml) and existing ontologies (dc, foaf, sioc). You can use
this as a template: http://esw.w3.org/topic/SIOC/RelatedOntologies

You're doing an excellent work, keep going! :-)
